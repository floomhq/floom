# Support

Floom uses **GitHub issues** for public repo support and **email** for private matters.

## Public support

Open a GitHub issue for anything that can be discussed in public:

- Bugs and regressions
- Docs corrections
- Feature requests
- Product or API behavior that needs clarification in the open

Start here:

- [Open issues](https://github.com/floomhq/floom/issues)
- [Bug report template](https://github.com/floomhq/floom/issues/new?template=bug_report.md)
- [Feature request template](https://github.com/floomhq/floom/issues/new?template=feature_request.md)

This repo does **not** use GitHub Discussions right now. Public questions go through issues so the answers stay searchable next to the code and docs.

Before opening an issue, check:

- [README.md](./README.md)
- [docs/README.md](./docs/README.md)
- [docs/SELF_HOST.md](./docs/SELF_HOST.md)
- Existing [GitHub issues](https://github.com/floomhq/floom/issues)

If you are reporting a bug, include the same details the bug template asks for:

- Floom version, image tag, or commit SHA
- Whether the problem is on `floom.dev` or a self-hosted deployment
- Relevant logs, screenshots, and reproduction steps

## Private support

Email [team@floom.dev](mailto:team@floom.dev) for anything that should not be posted in a public issue, including:

- Security vulnerabilities
- Code of Conduct reports
- Privacy or account-related requests
- Reproductions that require sharing secrets, tokens, or private deployment details

Related policies:

- [SECURITY.md](./SECURITY.md)
- [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)

## Contributing
