# Interview: Jan

- **Date**: 2026-04-03
- **Role**: CTO / technical co-founder, preparing fundraise
- **Duration**: ~20min

## Background

- Building something between Lovable and Claude Code
- Preparing for first trade fair in ~3 weeks
- Looking for technical people for the team
- Company: Apex Signal (apex-signal.tech)

## Current Stack

- Supabase edge functions for product workflows (news classification, matching, etc.)
- Notion for task management, knowledge management, meeting transcripts
- Previously used n8n for product workflows, migrated to edge functions for reliability/security
- Claude Code and Claude for company workflows
- Notion AI for meeting summaries, action items, workspace-wide search

## How They Work

- Two dimensions: **product** (edge functions in Supabase) and **company** (Notion + Claude)
- Product workflows: chain of edge functions processing news data (classify, match, etc.)
- Company workflows: Notion-based with AI features (transcription, summaries, task extraction)
- n8n was used for product but found it couldn't handle complex processes
- Not much repetitive work yet, still in creative ideation phase

## Feedback on Floom

- **Sees the deployment simplification value**: making it easy to deploy agents/workflows is clearly needed
- **Struggles to see differentiation from Claude/ChatGPT**: "if I let my father use ChatGPT... that already gets people very far"
- **Questions whether a full deployed frontend is always needed**: "would I need always a front end which is fully deployed, which I can share? Not sure"
- **Sees value in sharing with non-technical users**: agrees that sharing a Claude skill with non-technical people is hard
- **Wants integrations**: output is useful but "now I have to do something with the output" - wants deep integration (Notion, Slack bot, etc.)
- **Opportunity cost argument resonates**: "is it worth spending 20-30 minutes building the workflow? If you give me a tool which does this very solid, very fast, then I can see the value"
- **Wants use cases that go beyond what Claude can do alone**: needs to see clear value over just using Claude with skills

## Key Concerns

- Not clear where Floom sits vs. Claude Code skills, ChatGPT, etc.
- Integrations are critical (v2 isn't soon enough?)
- For one-time queries, doesn't see the fit
- Needs repeatable workflows to justify the tool

## Key Insight

- The value prop clicks when framed as: **deploy agents for non-technical users to consume**, not just "easier deployment for developers". The sharing/distribution angle is what differentiates from Claude Code skills.

## ICP Assessment

- Not a strong ICP fit currently - still in ideation phase, not much repetitive work
- End-of-call note: "I don't think he's an ICP"

## Action Items

- [ ] Sharpen positioning vs. Claude Code skills / ChatGPT for technical users
- [ ] Integrations story needs to be clearer earlier in the pitch
- [ ] Focus demo on repeatable workflows with non-technical end users
