# Interview: Jan-Ole
- **Date**: 2026-04-03

## Stack
- No N8N
- Lovable + Claude Code
- Notion (with Notion AI for transcribe -> parsing)
- Edge functions on Supabase

## Reaction to Floom

- Doesn't see the need as very important right now - doesn't have many repetitive workflows
- Would like it as an **invisible layer**: just make deployments work faster, behind the scenes
- Not looking for a workflow UI or orchestration tool - wants floom to disappear into the stack

## Key Signal

- "Invisible layer" framing: the product shouldn't feel like a tool you use, it should feel like infrastructure that makes everything else faster
- Lovable + Claude Code user = vibe-coder profile, not ops/automation heavy
- Supabase edge functions = already has a deployment path, floom competes here

## Positioning Gap

- N8N replacement pitch doesn't land for this persona (they never used N8N)
- The "instant deployment" angle is more relevant than "automation/cron"
