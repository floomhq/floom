# Interview: Liam

- **Date**: 2026-04-03
- **Role**: Employee at prop-tech company, n8n user
- **Duration**: ~15min

## Background

- Works at a prop-tech company (property management software)
- Built outbound automations using n8n (email, WhatsApp outreach)
- Also used Diffie for outreach workflows
- Connected tools: Slack, HubSpot, Front (email)
- Currently building an n8n workflow for internal churn process

## Current Workflow (Churn Process)

- Triggered when a client churns
- Sends push notification via Slack
- Pulls necessary data from HubSpot (client name, address, documents)
- Prepares email drafts, document links, to-do list
- Notifies team of physical documents to send

## Pain Points

- n8n is too complex for non-technical colleagues
- Many people at his company have n8n access but don't know how to use it
- Companies want to integrate AI but don't know how

## Demo Feedback

- Found the concept "super interesting"
- Gmail email prioritization use case: noted HubSpot already has some AI email intent detection built in
- Suggested a strong use case: companies have SOPs as written manuals or Loom videos explaining repetitive processes. A tool that can ingest these manuals/videos and auto-generate the automation workflow would be very sellable to large companies

## Key Idea

- **Manual/video-to-automation**: paste a process manual or transcribe a Loom video into the tool, and it generates and deploys the automated workflow. Removes the need for someone to understand n8n or code.

## n8n Ecosystem Notes

- Feels more people are moving away from n8n
- His colleague also uses it but it's not widely discussed
- Some teams connect n8n to Cursor to have AI generate workflows, but sees Floom as different from that approach

## Floom Fit

- Core value prop resonates: easy build + deploy in one go
- Biggest opportunity: SOP-to-automation pipeline for enterprises
- Current integrations (HubSpot, Slack) not yet available in Floom, but concept validated
